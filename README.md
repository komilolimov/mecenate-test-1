# Mecenate Feed App

Мобильное приложение для отображения ленты публикаций платформы Mecenate (аналог Patreon/Boosty). Разработано в рамках тестового задания.

## 🛠 Технологический стек

- **Платформа:** React Native + Expo (iOS / Android)
- **Язык:** TypeScript
- **Сетевой слой & Кэширование:** Axios + React Query (`@tanstack/react-query`)
- **Глобальное состояние:** MobX (`mobx`, `mobx-react-lite`)
- **UI & Стилизация:** Кастомный хук `useAppTheme` (Styles Factory + `useMemo`) для поддержки системной темной/светлой темы, SVG-иконки.
- **Архитектура:** Feature-Sliced Design (FSD) / Clean Architecture.

## 🚀 Как запустить локально

1. Клонируйте репозиторий:
   ```bash
   git clone [https://github.com/komilolimov/mecenate-test-1.git](https://github.com/komilolimov/mecenate-test-1.git)

Перейдите в директорию проекта и установите зависимости:
cd mecenate-test-1
npm install

Запустите проект:
npx expo start -c

Отсканируйте появившийся QR-код с помощью приложения Expo Go на вашем смартфоне (iOS/Android) или нажмите i для запуска в iOS-симуляторе.

✨ Реализованный функционал
Экран ленты (Feed)
[x] Лента постов: Отображение карточек с аватарами, текстом и обложками.

[x] Пагинация: Бесконечный скролл ленты (Infinite Queries).

[x] Фильтрация: Мгновенное переключение табов (Все / Бесплатные / Платные) через глобальный стор MobX.

[x] Платный контент: Заглушка с Blur-эффектом (expo-blur) для скрытия постов tier: "paid".

[x] Интерактив: Оптимистичное обновление UI (Optimistic Updates) при клике на лайк с использованием мутаций React Query.

[x] Обработка состояний: Кастомные UI-экраны загрузки, ошибки сети (с возможностью повтора) и пустого списка.

[x] Адаптивность: Полная поддержка переключения Dark/Light mode на уровне системы.

Детальный экран поста (Post Detail)
[x] Real-time обновления (WebSockets): Мгновенная синхронизация лайков и появление новых комментариев без перезагрузки экрана.

[x] Lazy Load комментариев: Порционная подгрузка списка комментариев по мере скролла.

[x] Отправка данных: Контролируемый инпут для написания комментариев с блокировкой интерфейса во время сетевого запроса.

[x] Анимации и Haptics: Плавная пульсация кнопки лайка (через Animated API) и тактильный отклик при нажатии.

[x] Оптимизация рендеринга: Строгая изоляция UI-компонентов (React.memo, useCallback) для предотвращения перерисовок тяжелых списков при вводе текста или получении данных по сокету.

[x] Pixel Perfect UI: Интеграция чистых векторных SVG-иконок напрямую из макета Figma.

🏗 Архитектура (Feature-Sliced Design)
Проект спроектирован с упором на масштабируемость и строгую изоляцию зон ответственности:

app/ — Инициализация приложения, глобальные провайдеры (React Query, Navigation).

pages/ — Композиция экранов (FeedPage, PostDetailPage).

features/ — Изолированная бизнес-логика (отправка комментариев, обработка лайков, фильтрация ленты, WebSocket-соединения).

entities/ — Бизнес-сущности (модели постов, карточки интерфейса).

shared/ — Переиспользуемый UI-kit (чистые компоненты без жестких классов фоновых цветов), API-клиент, дизайн-токены и конфигурация навигации.

⚙️ Переменные окружения (Environment Variables)
В рамках тестового задания API-ключи и базовые URL (Swagger/WS) настроены в клиенте напрямую (src/shared/api/client.ts). Дополнительных .env файлов для локального запуска не требуется.
